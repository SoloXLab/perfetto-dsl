# rules/

Global constraints that apply across workflows and skills.

## Files

- `github-agent-rules.md`: lifecycle, issue-first gate, branch/commit/PR/CI/release rules.
- `review-rules.md`: review focus, evidence expectations, and comment quality requirements.

## Usage

1. Read these rules before selecting workflow/skills.
2. Treat conflicts as: repository `AGENTS.md` > `rules/` > workflow/skill details.
