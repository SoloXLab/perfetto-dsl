# 用户诉求分析框架

## 三层诉求模型

分析用户评论时，始终尝试从三个层次理解用户诉求：

### 第一层：表层诉求（用户说的）

用户直接表达的功能需求或问题描述。
- 示例：“收不到消息通知”
- 示例：“加载太慢”
- 示例：“找不到设置在哪里”

### 第二层：场景诉求（用户在做什么）

用户当时处于什么使用场景，有什么目标。
- 示例：“等待重要工作消息时，怕错过”
- 示例：“赶时间付款时，页面卡死”
- 示例：“想调整隐私设置，找了很久”

### 第三层：情感诉求（用户真正需要什么）

用户内心深处的心理需求，通常是普世的人类需求。
- **安全感**：我的数据/钱/消息是安全的
- **掌控感**：我知道发生了什么，我能控制
- **效率感**：我能快速完成我想做的事
- **连接感**：我能与重要的人保持联系
- **信任感**：这个产品/品牌是可靠的
- **尊重感**：我的时间和注意力被尊重

## 常见产品类型的核心旅程

### IM / 通讯类

```text
收到通知 -> 打开 App -> 找到对话 -> 查看消息内容 -> 回复 -> 确认对方收到
```

关键断点高发节点：收到通知、找到对话、确认送达

### 电商类

```text
发现商品 -> 了解详情 -> 加购/收藏 -> 下单 -> 支付 -> 等待物流 -> 收货 -> 售后
```

关键断点高发节点：支付、等待物流、售后

### 内容/社区类

```text
进入 App -> 首页信息流 -> 发现感兴趣内容 -> 深度阅读/互动 -> 分享/收藏 -> 再次回访
```

关键断点高发节点：首页信息流质量、互动反馈

### 工具类

```text
明确需求 -> 找到功能入口 -> 执行操作 -> 等待结果 -> 使用结果
```

关键断点高发节点：找到功能入口、执行操作的复杂度

## 情感强度判断标准

| 强度 | 信号词（中文） | 信号词（英文） |
|---|---|---|
| 极强负面 | 垃圾、恶心、骗人、卸载、投诉、退款 | terrible, scam, uninstall, refund, disgusting |
| 强负面 | 很差、失望、不好用、坑、浪费 | horrible, disappointed, waste, frustrating |
| 中等负面 | 不方便、有点问题、希望改进 | inconvenient, issue, could be better |
| 中性 | 还行、一般、凑合 | ok, fine, average |
| 正面 | 好用、方便、喜欢、推荐 | useful, convenient, like, recommend |
| 强正面 | 非常好、超出预期、离不开、必备 | excellent, amazing, can't live without |

## 旅程图绘制模板（Markdown）

```markdown
| 旅程阶段 | 用户行为 | 用户心理 | 情感状态 | 体验评估 |
|---|---|---|---|---|
| 触发需求 | 收到重要联系人的消息 | "希望第一时间看到" | 😊 期待 | 🟢 |
| 寻找入口 | 等待通知弹出 | "通知怎么还没来？" | 😟 焦虑 | 🔴 断点 |
| 执行操作 | 手动打开 App 检查 | "只能自己去看了" | 😡 受挫 | 🔴 断点 |
| 获得结果 | 看到消息，但已延迟 | "消息发了很久了" | 😡 愤怒 | 🔴 断点 |
| 后续行为 | 抱怨 / 给差评 / 告知对方换其他方式联系 | "这个 App 不可靠" | 😡 流失风险 | 🔴 |
```
