# Market Discovery（发现机会）

## 技能概述

本技能完成产品立项前的「发现机会」阶段，回答一个核心问题：

> **这个市场值不值得进？如果进，凭什么能赢？**

分五节完成，每节结论直接作为下一节的输入：

```text
节一：用户需求历史回顾（需求演化螺旋）
  → 找到主要矛盾和机会入口
        ↓
节二：时机验证
  → 判断窗口是否开启、能持续多久
        ↓
节三：竞品深度分析
  → 用真实市场验证节一的主要矛盾
        ↓
节四：解决方案主轴确立
  → 确定我们的切入点、底层假设、目标用户
        ↓
节五：产品规划
  → 用户画像 / 定位图 / SWOT / 卖点 / 行为设计 / 变现 / 用户旅程
        ↓
发现机会 · 决策门禁
  → 输出：值得进 / 有障碍 / 时机未到 / 建议放弃
```

---

## 启动流程

收到用户输入后，判断从哪个节点开始：

- 只有产品概念 → 从**节一**完整开始
- 已有行业判断，想验证时机 → 从**节二**开始
- 已有竞品清单，想深度分析 → 从**节三**开始
- 已完成市场分析，想确立方向 → 从**节四**开始

如用户未说明，默认从节一开始，告知流程并确认。

## 节一：用户需求历史回顾

**分析目标：** 用需求生命周期螺旋结构，挖掘每个阶段的主要矛盾——需求是怎么萌芽、发展、成熟，又因为什么矛盾推动了下一个范式的出现。最后一个未解决的主要矛盾，就是我们产品的机会入口。

> ⚠️ 分析主体始终是**用户需求本身**，产品和技术只是需求演化的证据。不是产品时间轴，不是功能演化史。

### 核心框架：需求演化螺旋

任何行业的需求演化，都遵循同一个螺旋结构。每一轮循环内部经历四个阶段，循环结束时积累的主要矛盾触发下一轮更高层次的循环：

```text
第N轮循环
  ├─ 有无阶段：能不能用？（存在性问题）
  ├─ 成本阶段：用得起吗？（普及性问题）
  ├─ 体验阶段：用得好吗？（质量性问题）
  └─ 锁定阶段：走得了吗？（护城河问题）
        ↓
  【主要矛盾积累】
  现有范式把什么隐性成本转嫁给了用户？
  这个成本在现有范式内无法解决
        ↓
第N+1轮循环（更高层次的有无问题）
```

> **关键规律：每一轮循环解决了上一轮的主要矛盾，但把一个新的隐性成本转嫁给了用户。找到「被转嫁的隐性成本」，就找到了下一轮循环的起点。**

### 每个阶段的分析任务

**① 有无阶段**
- 需求是怎么被发现或命名的？是用户自己意识到，还是被外部事件/研究唤醒？
- 最早有这个需求的是哪类人？处境是什么？
- 第一个解法解决了什么存在性问题？

**② 成本阶段**
- 什么力量推动了从小众到大众的普及？
- 成本障碍是价格、效率还是可及性？谁用什么方式打破了它？

**③ 体验阶段**
- 谁赢得了市场第一？用什么方式赢的？
- ⚠️ 市场排名是验证工具——第一名一定命中了用户需求的某一层，找到那一层
- 是质变（底层逻辑跃迁）还是量变（更好实现同一逻辑）？
- 后来的细分背后是哪类用户的哪种强诉求？

**④ 锁定阶段**
- 现有主流解法用什么机制锁住用户？
- 失效信号是什么？（差评趋势、留存下降、用户抱怨主题）
- 失效是执行问题（可修复）还是范式天花板（无法在现有框架内解决）？

### 挖掘每个阶段的主要矛盾

> **主要矛盾** = 这个阶段「用户真实需求」和「现有解法能提供的」之间最核心的张力。正是这个张力，驱动需求演化到下一阶段。

根据矛盾性质选择分析工具（不写死，根据实际判断）：

```text
矛盾性质                          工具选择
────────────────────────────────────────────────────
根因是单一链条                  → 5 Whys
根因来自多个独立方向            → 鱼骨图（用户/解法/外部环境/商业模式）
核心是用户行为/动机的矛盾      → JTBD 三层动机
  功能层：用户想完成什么
  情感层：完成时想感受到什么
  社会层：完成后想让别人怎么看自己
  → 找出解法满足的层次 vs 用户真正卡住的层次
矛盾是系统性的、循环的          → 系统思考
  增强回路：让问题越来越严重的正反馈
  调节回路：为什么干预只能短期有效
矛盾涉及多方利益冲突            → 利益相关者矩阵
  区分「这家公司的选择」vs「结构性约束——换了谁都会面临」

+ 任何矛盾结论最后必做反驳测试：
  「如果这个判断是错的，应该能看到什么反例？现实中有没有？」
```

**主要矛盾输出格式：**

```text
【主要矛盾】
用户真实需要：[...]
现有解法提供的：[...]
核心张力：[一句话说清两者之间的根本差距]
转嫁的隐性成本：[现有解法把什么负担转嫁给了用户]
使用工具：[选了哪个工具，为什么]
反驳测试：[如果判断错了，应该看到什么反例？]
驱动力：→ [这个矛盾如何触发下一轮循环]
```

**完整输出格式：**

```text
## 第N轮循环：[范式名称]（[时间段]）

### 有无阶段
[分析]

### 成本阶段
[分析]

### 体验阶段
[分析]

### 锁定阶段
[分析]

【主要矛盾】
用户真实需要：...
现有解法提供的：...
核心张力：...
转嫁的隐性成本：...
使用工具：...
反驳测试：...
驱动力：→ 触发第N+1轮循环
```

最后输出**需求演化螺旋结论表**：

| 轮次 | 范式 | 核心矛盾 | 有无 | 成本 | 体验 | 锁定 | 转嫁的隐性成本 |
|---|---|---|---|---|---|---|---|
| 第一轮 | | | | | | | |
| 第二轮 | | | | | | | |
| 第N轮 | | | | | **现在所处** | | |
| 下一轮 | | **← 机会入口** | 尚无／初现 | - | - | - | 待观察 |

> ⚠️ **结论表最后一行最重要**——下一轮循环的「有无问题」，直接指向我们产品需要解决的核心问题。后续所有分析以这个结论为起点。

## 节二：时机验证

**分析目标：** 基于节一推导出的「下一轮循环机会入口」，客观评估现在进入这个市场的时机——不是为进入找理由，而是如实判断窗口在哪里、有多长、适不适合现在进。

结论可能是「现在是最佳窗口」，也可能是「时机未到」或「窗口已过」。

### 第一步：定位当前所处位置

根据节一的螺旋结论，下一轮循环现在处于哪个阶段？

```text
有无阶段（早期）  尚无主流解法，需求刚被命名，用户还没有成熟认知
有无阶段（中期）  少数先行者在探索，但没有人真正解决存在性问题
有无阶段（末期）  解法雏形出现，竞争者开始涌入，窗口快关闭
成本阶段          有无问题已被解决，竞争转向规模化和价格
体验阶段          存量博弈，差异化是唯一出路
锁定阶段          市场成熟，进入成本极高
```

明确标注：**我们现在处于 [____] 阶段**，判断依据是 [____]。

### 第二步：客观评估加速和阻碍力量

加速力量（推动新范式更快到来）：
- 每条标注确定性：⭐⭐⭐高 / ⭐⭐中 / ⭐低

阻碍力量（让现有范式延续更久，或让新范式更难落地）：
- 每条标注严重程度：🔴 致命 / 🟡 重要 / 🟢 可克服

重点识别：多条加速力量交叉叠加的位置，那里是窗口最强的地方。

### 第三步：时机窗口评估

```text
当前阶段：[有无早期 / 有无中期 / 有无末期 / 成本阶段 / ...]
窗口开启时间：[已开启 / 预计X年后开启]
窗口持续时长：[预计X年，依据是...]
先发优势价值：[高/中/低，原因是...]
最大风险：[什么情况会让窗口提前关闭或根本无法打开]
```

### 节二结论：客观时机判断

基于以上分析，得出以下四种结论之一：

```text
✅ 现在是最佳进入窗口
   依据：[核心理由]
   建议姿势：[对应这个阶段的进入策略]

⚠️ 窗口存在，但有关键障碍
   依据：[窗口存在的理由]
   障碍：[什么条件还不成熟]
   建议：[等待什么条件成熟，或用什么方式绕过障碍]

🕐 时机未到，窗口尚未开启
   依据：[什么条件还缺失]
   预计窗口开启时间：[X年后，触发条件是...]
   现在可以做什么：[提前布局 / 持续观察]

❌ 窗口已过，或这个市场不值得进
   依据：[为什么错过了，或为什么不值得]
   替代方向：[如果仍想进入，有没有其他切入点]
```

这个结论直接决定后续分析是否继续——如果结论是❌，后续竞品分析和产品定义可以终止。

## 节三：竞品深度分析

**分析目标：** 用真实竞品验证节一推导出的主要矛盾——在现实市场上是否成立，有没有人已经在解决，解决到什么程度。节三是节一的市场验证，不是孤立的竞品扫描。

> ⚠️ 如果节二结论是❌，本节可以终止。

### 前置：带着节一的结论进入

在开始任何竞品分析之前，先明确两个问题：

```text
节一主要矛盾是：[从节一结论直接复制]
被转嫁给用户的隐性成本是：[从节一结论直接复制]

带着这两个问题审视每一个竞品：
① 这个竞品解决了这个矛盾吗？解决到什么程度？（用百分比估算）
② 它把什么新的隐性成本转嫁给了用户？
③ 它的用户声音，是否在印证这个矛盾的真实存在？
```

### 步骤一：竞品识别

如用户未提供竞品，执行以下搜索策略：

```text
1. "[产品类别] app 竞品 排行 [当前年份]"
2. "[产品类别] best apps alternatives [当前年份]"
3. "App Store / Google Play [类别] top charts"
4. "site:reddit.com [产品类别] app recommendation"
5. "知乎 OR 少数派 [产品类别] app 推荐"
```

搜索完成后，向用户列出推荐竞品清单，分三层：

| 层级 | 数量 | 分析深度 |
|---|---|---|
| **直接竞品** | 3-5个，同赛道同目标用户 | 七个维度完整分析 |
| **间接竞品** | 2-3个，不同方式解决同一问题 | 三个维度简要分析 |
| **参照产品** | 1-2个，商业模式或设计值得借鉴 | 商业模式 + 启示 |

询问用户确认后开始分析。

### 步骤二：直接竞品分析（七个维度）

**维度一：产品定位**
- 一句话定位
- 核心差异化是什么
- 目标用户是谁
- 当前所处生命周期（冷启动 / 成长 / 成熟 / 衰退）
- **对节一主要矛盾的解决程度：[X%，说明解决了什么，没解决什么]**

**维度二：AARRR 增长分析**

**Acquisition 拉新**
- 主要流量来源（ASO / 社区口碑 / KOL / 付费买量）
- ASO 关键词策略（覆盖哪些词？）
- 社区种草情况（Reddit / 知乎 / 小红书有无大量用户自发推荐）

**Activation 激活**
- 用户的"啊哈时刻"是什么（第一次感受到核心价值的瞬间）
- 新手引导设计（几步？门槛高不高？）
- 从下载到体验核心功能需要几分钟

**Retention 留存**
- 第2/7/30天靠什么机制留住用户
- 习惯养成设计（连续打卡 / 成长系统 / 社交绑定）

**Revenue 变现**
- 付费墙在哪里，付费时机设计
- 变现模型（买断 / 订阅 / 内容包 / 广告 / 混合）
- 价格点和套餐结构

**Referral 传播**
- 社交货币设计（什么值得分享）
- 用户主动传播的场景

**僵尸用户激活**
- 召回机制（Push通知 / 邮件 / 限时活动）

**维度三：变现阶段分析**

判断竞品当前处于哪个变现阶段，分析策略是否与阶段匹配：

| 阶段 | 典型变现方式 | 匹配判断标准 |
|---|---|---|
| 冷启动期 | 免费 / 极低价买断 | 是否最小化付费摩擦 |
| 成长期 | 买断 + 内容包 | 是否在建立付费习惯 |
| 规模期 | Freemium + 可选订阅 | 订阅是否为附加层 |
| 成熟期 | 订阅为主 + B2B | 是否有第二曲线 |
| ⚠️衰退信号 | 强推订阅 / 功能锁订阅 / 涨价 | 这是竞品失误信号，意味着市场出现空白 |

**维度四：用户真实声音**

必须搜索真实评论，不能凭空描述：

```text
1. "[竞品名] reviews site:reddit.com"
2. "[竞品名] 差评 OR 用户评价 site:zhihu.com"
3. "[竞品名] Google Play reviews"
4. "[竞品名] App Store negative reviews"
5. "[竞品名] 少数派 OR sspai 评测"
```

对收集到的评论做三类聚类，每类标注频率：

**好评聚类**：用户因为什么选择 / 留下来 / 愿意付费 / 主动推荐

**差评聚类**：用户因为什么愤怒 / 离开 / 退款 / 在社区发泄

**中评聚类**：用户想要但没得到什么 / 接受但不满意什么

每条聚类主题附：典型原话（标注来源）+ 频率评级：⭐⭐⭐⭐⭐极高 / ⭐⭐⭐⭐高 / ⭐⭐⭐中 / ⭐⭐低

**聚类完成后，强制问：**
> 这些差评和中评，背后是否都在指向同一个被转嫁的隐性成本？
> 是否在印证节一的主要矛盾？

**维度五：用户群体识别**

从评论中识别出不同细分用户群，分析各群核心诉求差异：

```text
细分群 A：[群体标签]
  核心场景：...
  核心诉求：...
  付费意愿：...
  对节一主要矛盾的感受：[这类用户对这个矛盾的痛感有多深？]
```

**维度六：核心矛盾与深层动机**

**6-A 核心矛盾识别**

格式：「用户最爱这个产品的 [X]，但产品的 [Y] 正在损耗这个爱。」

然后问：这个竞品的核心矛盾，是不是节一主要矛盾的一个具体体现？

**6-B 深层动机挖掘**（必做项，不能跳过）

触发条件：竞品出现以下任何情况必须执行：
- 商业模式转变
- 功能大幅削减或增加
- 用户口碑短期内显著变化
- 某个用户行为反常
- 某个问题长期存在但竞品没有解决

根据现象类型选择分析工具：

```text
现象类型                      工具
──────────────────────────────────────────────
根因是单一链条              → 5 Whys
根因来自多个独立方向        → 鱼骨图
核心是用户行为/动机         → JTBD 三层动机
  功能层：用户想完成什么
  情感层：完成时想感受到什么
  社会层：完成后想让别人怎么看自己
  ⚠️ 差异化空间在情感层和社会层，不在功能层
竞品团队的决策行为          → 利益相关者矩阵
  区分「这家公司的选择」vs「结构性约束——换了谁都会面临」
问题长期存在、循环反弹      → 系统思考
  找增强回路和调节回路
```

任何结论必做反驳测试：
> 「如果这个解释是错的，应该能观察到什么反例？」

输出格式：

```text
【深层动机分析】[竞品名] 的 [具体现象]
▌使用工具：[工具名] + [选择原因]
▌分析过程：[按所选工具展开]
▌根因结论：[区分选择 vs 结构性约束]
▌反驳测试：[反例存在吗？]
▌对我们的压力测试：同样的约束何时作用到我们身上？
```

**维度七：对我们的启示**

- 可以直接借鉴的机制
- 必须避免的失误（区分「主动失误」vs「结构性约束导致的必然」）
- 可以承接的失望用户群
- **这个竞品印证了节一主要矛盾的哪个具体方面？**

### 步骤三：间接竞品分析（三个维度）

1. Revenue 变现模型
2. 用户声音（好评 + 差评，各3条以上真实原话）

## 节四：解决方案主轴确立

**分析目标：** 在进入产品规划之前，先从节一的主要矛盾出发，确立我们产品的核心解决方向——所有后续产品决策（用户、定位、卖点、行为设计、变现）都以这个方向为主轴推导，不能偏离。

### 前置：带着三个节点的结论进入

```text
节一主要矛盾：[复制节一结论]
被转嫁给用户的隐性成本：[复制节一结论]
节三验证结论：✅成立 / ⚠️部分成立（复制节三汇总三的结论）
竞品解决度最高：X%（复制节三主要矛盾验证图）
```

### 第一步：确立解决方向

回答三个问题：

```text
① 我们打算解决这个矛盾的哪个层次？
   全部解决（野心大，风险高，资源要求高）
   解决最核心的一个切入点（聚焦，更容易验证）
   → 建议：选一个切入点，说明为什么是这个而不是别的

② 我们用什么底层逻辑解决？
   和所有竞品的底层假设是什么？
   我们的底层假设是什么？
   → 两者必须不同，否则我们只是「更好的竞品」，而不是「新范式」

③ 我们解决这个矛盾的核心手段是什么？
   是技术差异（别人做不到的能力）？
   是设计差异（别人没想到的方式）？
   是商业模式差异（别人不愿意做的取舍）？
   → 这个手段必须是竞品在短期内无法复制的
```

### 第二步：目标用户分群与筛选

不是「谁需要这个产品」，而是「谁被这个主要矛盾卡得最深」。

用户分群分三步走：先收集原始信号，再聚类，再用 Checklist 筛选。

**2-A：收集原始信号**

```text
竞品评论        → Google Play / App Store / 亚马逊评论
社区讨论        → Reddit / 知乎 / 小红书 / 豆瓣
社交媒体        → Twitter/X / TikTok 用户自发内容
行业报告        → 已有分群数据作为参照
```

收集重点：**用户在什么处境下出现了这个需求？他们原来用什么解决？为什么不满意？**

**2-B：沿两个轴聚类**

- 轴一：使用场景（场景不同 → 需求本质不同 → 是不同的用户群）
- 轴二：核心动机（JTBD 三层：功能层/情感层/社会层；功能层相同但情感层不同 → 是不同的用户群）

**2-C：验证分群有效性**（内部一致性 / 外部差异性 / 可操作性，三个都满足才有效）

**2-D：用 Checklist 筛选目标群**

| 分群 | ①矛盾关联(1-5) | ②竞品满足↓(1-5) | ③LTV(1-5) | ④获取(1-5) | ⑤验证(Y/N) | 结论 |
|---|---|---|---|---|---|---|
| [分群A] | | | | | | |

输出结论：

```text
种子用户：[得分最高的群，说明原因]
主力市场：[规模最大的群]
隐藏机会：[竞品满足度极低但被忽视的群]
放弃：    [与主要矛盾关联弱，或竞品已充分满足的群]
```

### 第三步：卖点主轴确认

```text
我们的第一卖点，是否直接指向节一的主要矛盾？
→ 是：这是我们最强的护城河，继续
→ 否：重新审视卖点排序，可能出了问题
```

### 第四步：行为设计主轴确认

```text
节一主要矛盾转嫁给用户的隐性成本是：[...]
行为设计要强化的核心感受是：[情感层和社会层的具体描述]
后续所有行为设计步骤，都要能回答「这个设计如何强化这个核心感受」
```

### 节四输出：解决方案主轴

```text
主要矛盾：[一句话]
我们的解决切入点：[一句话]
底层假设（我们 vs 竞品）：
  竞品假设：[...]
  我们的假设：[...]
核心手段：[技术/设计/商业模式差异，一句话]
目标用户：[被矛盾卡得最深的那群人，一句话]
第一卖点：[直接指向主要矛盾，一句话]
行为设计核心感受：[情感层+社会层，一句话]
```

> ⚠️ 这个输出是节五产品规划的起点。节五所有决策必须能回答：「这个决策如何帮助我们更好地解决主要矛盾？」

## 节五：产品规划

**前置：承接节四的解决方案主轴，所有规划以节四输出为起点，不重新推导。**

### 5.1 目标用户画像

直接从节四的分群结论展开，对种子用户、主力市场、隐藏机会分别做深度画像。

每个画像必须用 JTBD 三层动机挖掘：
- 功能层（表层，竞品都在满足）
- 情感层（中层，差评中隐藏，竞品常忽视）
- 社会层（深层，用户自己也很少说出口）

每个画像输出：人物标签 / 核心使用场景 / JTBD三层动机 / 现有解决方案及其不满（竞品差评原话支撑）/ 对我们产品的核心期待 / 付费意愿与心理价格点

### 5.2 竞品定位图

- 来自用户真正在乎的（从好评差评中提炼，不能是PM自己设定）

标出所有竞品位置 + 我们的目标位置，说明为什么这个象限是空白或值得占领。

### 5.3 SWOT 分析

每条 SWOT 项必须标注来源（行业洞察 / 竞品分析 / 趋势判断）。

| 优先级 | 卖点 | 支撑能力 | 为什么竞品没有 | 用户感知方式 |
|---|---|---|---|---|
| 🥇 | [节四第一卖点] | | | |
| 🥈 | | | | |
| 🥉 | | | | |

排序后强制验证：第一卖点是否直接指向主要矛盾？否则返回节四。

### 5.5 行为设计规划

基于 Hooked Model + Flow Theory + Progression Systems + 行为神经科学。

> ⚠️ 核心原则：**正向训练，不对抗**。所有设计步骤的主轴：强化节四确定的「核心感受」。

六个步骤：
1. 识别用户放弃根因（情绪触发 / 外部干扰 / 启动摩擦 / 奖励感不足），针对性设计
2. 激活设计（首次体验必须快速给成功感）
3. 使用深度自动拉伸（动态适应，不强制，早/中/后三阶段）
4. 奖励系统设计（基础/随机/里程碑/惊喜四层奖励，防止适应）
5. 社交机制设计（他人在场效应 / 进度对比 / 成就展示 / 社群归属）
6. 成长可见性设计（核心成长指标必须和「核心感受」直接相关）

### 5.6 变现路线图

**必须与产品生命周期严格对应，不得跨阶段使用变现策略。**

| 阶段 | 阶段目标 | 核心变现方式 | 设计逻辑 | 风险红线 |
|---|---|---|---|---|
| **冷启动（0→1）** | 获取种子用户，验证核心体验 | 买断解锁（低价一次性） | 最低摩擦，筛选高意愿用户 | 不得在首次体验前触发付费 |
| **成长期（1→10）** | 建立付费习惯，提升 LTV | 买断 + 内容包矩阵 | 老用户复购，内容包定价低于核心买断 | 内容包不能包含核心功能 |
| **规模期（10→100）** | 稳定 ARR，降低新用户依赖 | 可选订阅（附加层） | 订阅永远不覆盖已买断功能 | 不得将已买断功能迁移至订阅 |
| **成熟期** | 第二曲线，多元收入 | B2B 学校/企业批量授权 | 高 LTV，续约率高 | 不能稀释 C 端产品口碑 |

变现设计原则（不可违反）：买断功能永不迁移订阅 / 付费触发在价值感知之后 / 僵尸用户激活用折扣不用惩罚。

## 特殊场景处理

**用户只有一句话的产品概念** → 用3-5句话复述理解，确认方向后再开始

**用户已有竞品清单** → 直接用用户提供的清单，追加搜索用户未提但重要的竞品

**竞品是海外产品，中文资料少** → 优先搜索 Reddit、Product Hunt、App Store 英区评论

**产品是 B2B / 企业软件** → AARRR 中 Referral 改为 Expansion，变现重点分析定价矩阵和合同结构

**用户想跳过某个节** → 告知跳过的风险，尊重用户决定，在输出中标注「基于假设，建议后续验证」
