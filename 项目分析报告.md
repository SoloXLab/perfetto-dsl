# Perfetto DSL 项目分析报告

## 📋 项目概述

**Perfetto DSL** 是一个为 Perfetto trace 文件提供链式 DSL（领域特定语言）访问的 Python 库。它提供了直观、流畅的 API 来查询和分析 Perfetto 性能追踪数据，支持懒加载、关联对象属性访问、强大的参数过滤功能和高级 slice 定位语法。

### 基本信息
- **项目名称**: perfetto-trace-dsl
- **版本**: 1.0.0
- **许可证**: MIT
- **Python 版本要求**: >= 3.8
- **主要依赖**: perfetto>=0.14.0

### 核心模块分析

#### 1. **Trace 类** (`trace.py`)
- **职责**: 项目的主入口点，管理 TraceProcessor 生命周期
- **功能**:
  - 使用上下文管理器（`with` 语句）确保资源正确释放
  - 提供统一的查询入口：`slice()`, `counter()`, `cpu_freq()`, `cpu_usage()`, `metric()`
- **设计模式**: 门面模式（Facade Pattern）

#### 2. **SliceQueryBuilder** (`slice_query_builder.py`)
- **规模**: 3367 行代码，是项目的核心模块
- **核心类**:
  - `Slice`: 表示 Perfetto 中的 slice 数据
  - `Track`: 表示 track 数据
  - `Flow`: 表示 flow 关系
  - `RelatedObjectsAccessor`: 关联对象访问器
  - `ArgsValues`: Args 值集合，支持统计操作
  - `SliceQueryBuilder`: 查询构建器
- **核心功能**:
  - 链式查询构建
  - 高级 slice 定位语法：`slice_name|process_name||thread_name|||track_name#index`
  - Args 参数过滤和统计
  - Flow 关联查询
  - 时间范围查询（before, after, between）
  - 聚合统计（count, max, min, avg, p90, p95）
  - 模糊匹配（SQL LIKE）

#### 3. **CpuUsageQueryBuilder** (`cpu_usage_query_builder.py`)
- **核心类**:
  - `CpuUsageData`: CPU 使用率数据类
  - `CpuUsageQueryBuilder`: CPU 使用率查询构建器
- **功能**:
  - 整体 CPU 使用率查询（返回小数格式，1.0=100%）
  - 进程级 CPU 使用率（字典格式，按使用率排序）
  - 线程级 CPU 使用率（字典格式，按使用率排序）
  - 进程-线程嵌套字典
  - 时间范围过滤
  - 进程/线程过滤

#### 4. **CpuFreqQueryBuilder** (`cpu_freq_query_builder.py`)
- **核心类**:
  - `CpuFreqData`: CPU 频率数据类
  - `CpuFreqQueryBuilder`: CPU 频率查询构建器
- **功能**:
  - CPU 频点字典（不同 CPU 核心上不同频点的运行时长）
  - 进程分组（显示不同进程在不同 CPU 上的频点运行时长）
  - 线程分组
  - 进程-线程嵌套分组
  - CPU 核心分组（如 little/big 核心分组）
  - 时间范围过滤

#### 5. **CounterQueryBuilder** (`counter_query_builder.py`)
- **核心类**:
  - `Counter`: Counter 数据类
  - `CounterQueryBuilder`: Counter 查询构建器
- **功能**: Counter 数据的查询和过滤

#### 6. **MetricQueryBuilder** (`metric_query_builder.py`)
- **核心类**: `MetricQueryBuilder`
- **功能**:
  - 列出所有可用的 metrics
  - 执行指定的 metric 查询

## ✨ 核心特性分析

### 1. 链式 DSL 设计
```python
# 示例：流畅的链式调用
slices = (trace.slice(name="%doFrame%")
          .process(name="app")
          .thread(name="main")
          .after(playing_slice)
          .args("Layer name", "Surface")
          .all())
```

**优势**:
- 代码可读性强，接近自然语言
- 支持方法链式组合，灵活构建复杂查询
- 懒加载机制，只在需要时执行查询

### 2. 高级 Slice 定位语法
```
slice_name|process_name||thread_name|||track_name||||layer_name#index
```

**语法说明**:
- `|` 分隔 slice 名称和进程名
- `||` 分隔进程名和线程名
- `|||` 分隔线程名和 track 名
- `||||` 分隔 track 名和图层名
- `#` 指定索引（0=第一个，-1=最后一个）

**优势**:
- 精确匹配，避免多 slice 匹配问题
- 符号安全，避免与 slice 名称冲突
- 支持在时间范围查询中使用

### 3. Args 参数过滤系统
```python
# 获取 args
args_dict = slice_obj.args()
surface_token = slice_obj.args("Surface frame token")

# 过滤
slices = trace.slice().args("Surface frame token", 906975).all()

# 统计
surface_tokens = trace.slice().args("Surface frame token")
print(surface_tokens.count())
print(surface_tokens.max())
print(surface_tokens.unique())
```

**功能**:
- 支持精确匹配和模糊匹配
- 支持多个 args 参数组合查询
- 支持统计操作（count, max, min, avg, sum, unique_count, unique）
- 支持索引访问和迭代

### 4. 关联对象访问
```python
slice_obj = trace.slice(id=123).first()
process_name = slice_obj.get_process_name()
thread_name = slice_obj.get_thread_name()
track_name = slice_obj.get_track_name()
```

**实现方式**:
- 通过 `process_track` 和 `thread_track` 两种方式获取进程信息
- 智能信息显示，Slice 对象自动显示所有可用信息
- 缓存机制，避免重复查询

### 5. CPU 分析功能

#### CPU 使用率
- **返回格式**: 小数格式（1.0 = 100%），便于计算
- **分组方式**: 支持整体、进程、线程、进程-线程嵌套
- **排序**: 按使用率从高到低排序

#### CPU 频率
- **数据格式**: 字典格式，显示不同 CPU 核心上不同频点的总运行时长
- **分组功能**: 支持按进程、线程、CPU 核心分组
- **自定义分组**: 支持 little/big 核心分组或自定义分组

### 6. 懒加载机制
- 查询只在调用 `.all()`, `.first()`, `.count()` 等方法时执行
- 支持直接打印 QueryBuilder，自动执行查询并显示结果
- 提高性能，避免不必要的查询

## 🔧 技术实现细节

### SQL 查询构建
- 基于 Perfetto SQL 表结构设计
- 使用 CTE（Common Table Expression）优化复杂查询
- 支持多表 JOIN（slice, thread, process, track, args, flow）
- 错误处理：优雅处理数据不存在的情况

### 数据模型
- 使用 `@dataclass` 定义数据类，提供类型安全
- 支持字典和对象两种访问方式
- 自动单位转换（纳秒 ↔ 毫秒）

### 类型注解
- 完整的类型注解（typing 模块）
- 支持 Union、Optional、List、Dict 等类型
- 提高代码可读性和 IDE 支持

## 🎯 使用场景

### 适用场景
1. **Android 性能分析**: 分析 Android 应用的性能问题
2. **系统性能调优**: 分析系统级性能瓶颈
3. **CPU 性能分析**: 分析 CPU 使用率和频率分布
4. **帧率分析**: 分析渲染帧率和帧时间
5. **进程/线程分析**: 分析特定进程或线程的性能

### 典型用例
```python
# 1. 查找特定 slice
slice = trace.slice(name="doFrame|launcher").first()

# 2. 分析 CPU 使用率
overall = trace.cpu_usage()  # 整体使用率
process_usage = trace.cpu_usage().process()  # 进程使用率

# 3. 分析 CPU 频率
freq = trace.cpu_freq().group(little=[0,1,2,3], big=[4,5,6,7])

# 4. 复杂查询
slices = (trace.slice()
          .between("start|process", "end|process")
          .args("Layer name", "Surface")
          .all())
```
